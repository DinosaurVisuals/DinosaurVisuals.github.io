Lecture 1: Intro & Web Dev101

//Lecture Takeaway//
What is Web Development (Dev)? Web development refers to the creation, building and maintenance of websites or application that work over the internet. For biomedical communications purpose, the instructor break web dev into three core areas: (1) Backend dev (2) Frontend dev (3) Full-stack dev

  • Backend dev
    • The backbone of the web
    • Invisible to users but allow the app to function
    • Referred to as server side
    • Common technologies: PHP, Java, Python, Node.js, Ruby, C# and C++
  • Frontend dev
    • The face of the web
    • Interact with user, such se structure layout and behavior of the app
    • Referred to as a client side
    • Common technologies: HTML, CSS, Javascript
  • Full-stack dev
    • Backbone + Face
    • Deep understand of both areas
Website/ Web app vs. web browser:
  • Website is a server that host a file (e.g., Chrome, Safaria)
  • Web browser are programs load websites through an HTTP request to a server

Lecture 2: HTML and CSS

//Lecture Takeaway//
What is different between HTML, CSS and JavaScript?

  • HTML (HyperText Markup Language): Used to design the frontend of webpages using markup language and acts as the skeleton for a website
  • CSS (Cascading Style Sheets): Used to simplifying the process of making web pages look good by defining styles of a website or app
  • JavaScript: Used to provide dynamic behaviour to a website — in short, it adds functionality like animations and interactive features

Lecture 4: Preproduction for nonlinear media

//Lecture Takeaway//
What is the typical workflow for linear media? Treatment -> Script -> Concept Art -> Storyboard -> Prototype -> Production;
Notes from peer feedback on media lab in class:

  • Storyboard Enhancements:
    • Add sound effect notes, timing, transitions, and action cues.
    • Use color coding to separate camera moves and highlight key actions.
  • Camera & Motion Techniques:
    • Explore birds-eye views, camera following main objects for immersion, breaking arrows out of frames, and using dotted arrows for tunnels.
    • Hard cuts often work better
  • Text & Clarity:
    • Use different text elements (" ", < >) in description boxes, callout boxes for emphasis, and consistent highlight styles for action words.
    • Place scripts at the bottom and other text to the side for clarity.

// After Class Note //
The biggest takeaway is all the tips I learned for creating storyboards—how to use arrows effectively and how to make the story more engaging.

Lecture 5: Scrollytelling

//Lecture Takeaway//
What is Scrolly telling?
Scrollytelling utilizes the natural act of scrolling through a webpage or app to create an engaging storytelling experience. This technique allows communicators to transform long-form content into an interactive journey, making it more immersive and dynamic. By incorporating animated visuals and audio elements, scrollytelling enhances user engagement and deepens their connection to the narrative. It is particularly effective for linear stories with a clear chronological structure, guiding the audience seamlessly through the content while maintaining their interest.
